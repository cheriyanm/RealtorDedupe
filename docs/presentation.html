The problem

One agent. Many identities.

The same real estate agent appears differently across MLS systems with no single unique identifier.

MLS System A

Name: Thomas Johnson
Phone: 704-555-1234
License: NC-12345
Email: tjohnson@gmail.com

MLS System B

Name: Tom Johnson
Phone: (704) 555-1234
License: 12345
Email: tom.j@realty.com

Same agent — different name, different license format, different email. Traditional deduplication fails here.

The challenge

Why existing approaches failed

Deterministic rules

Too rigid. Breaks on Tom/Thomas, prefix variations, and phone formatting differences.

Levenshtein distance

String-only comparison. "Thomas" vs "Tom" scores low even though they're the same person.

Jaro-Winkler

Better but still no semantic understanding. Doesn't understand context across fields.

The core problem: no single field is reliable across all MLS systems. We need to combine signals intelligently.

The solution

A 5-step ML pipeline

5
Confidence decisions
Auto merge · LLM review · Keep separate — based on final score
Step 2 — Blocking

Smart grouping reduces comparisons by 99%

Instead of comparing every agent against every other agent, we group records into smart buckets first.

Without blocking

127,260

comparisons needed
(n × (n-1) ÷ 2)

With blocking

1,169

candidate pairs
99.1% reduction

Blocking rules: name prefix + phone last 7 digits + email username + license number (OR logic — any match puts records in the same bucket)

Step 3 — Embeddings

Semantic vectors understand meaning

Each agent record is converted to a list of 384 numbers that capture semantic meaning — not just characters.

Model used: all-MiniLM-L6-v2 — pre-trained on billions of text examples, understands name variations, abbreviations, and context.

Step 4 — Splink

Evidence vs coincidence

Splink weighs each field by how unlikely it is to match by chance — rare matches get higher weight.

+40 pts

Splink learns these weights automatically from your data using the Expectation-Maximisation algorithm. No manual tuning needed.
